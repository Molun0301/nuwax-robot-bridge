# nuwax_robot_bridge 接入 Nuwax Docker Client

Nuwax Docker Client 的通用部署步骤参考官方文档：

<https://nuwax.com/docker-client-deploy.html>

本文只补充当前 `nuwax_robot_bridge` 项目与 Nuwax Docker Client 对接时，必须关注的项目内配置和宿主机侧约束。

本文及项目仅以宇树Go2作为实现案例，宇树G1机器人以及其他品牌机器人的接入步骤类似。

## 模式选择

### 基础模式

默认按不启用 TTS 的方式理解本文。

建议在 `nuwax_robot_bridge/.env` 中保持：

```text
TTS_PLAYER_AUTO_START=false
TTS_LOG_BRIDGE_ENABLED=false
```

这种模式下，你可以跳过：

- Doubao 配置
- 机身扬声器 / 本地播放链路配置
- 宿主机 Nuwax 日志路径确认
- 日志桥接的 root 权限要求
- TTS 联调验证

### TTS 增强模式

如果你需要语音能力，再单独开启 TTS 相关配置。

常见组合：

- 只启用 TTS 运行时：`TTS_PLAYER_AUTO_START=true`，`TTS_LOG_BRIDGE_ENABLED=false`
- 启用 Nuwax 日志自动播报：`TTS_PLAYER_AUTO_START=true`，`TTS_LOG_BRIDGE_ENABLED=true`

启用日志播报时，链路如下：

```text
Nuwax Agent Client 容器
    -> /home/user/.nuwaxbot/logs/main.log
    -> Docker volume: nuwax_home
    -> 宿主机路径: /var/lib/docker/volumes/nuwax_home/_data/.nuwaxbot/logs/main.log
    -> nuwax_robot_bridge log_tts_bridge 监听该文件
    -> Doubao 实时 TTS
    -> Go2 机身扬声器 / 本地回退音频
```

当前项目 `.env` 已经按这个卷路径写好了默认值：

```text
TTS_LOG_BRIDGE_LOG_PATH=/var/lib/docker/volumes/nuwax_home/_data/.nuwaxbot/logs/main.log
```

所以这份部署文档默认你继续使用官方示例里的卷名 `nuwax_home`。如果你改了卷名，就必须同步修改 `nuwax_robot_bridge/.env`。

## 默认前提

默认你已经按官方文档把 Nuwax Docker Client 部署完成，并且容器本身工作正常。

`nuwax_robot_bridge` 这边额外需要满足：

- Go2 所在网络可从宿主机访问
- `nuwax_robot_bridge` 运行环境建议 Python `3.10+`
- 当前解释器已能导入 `unitree_sdk2py`

只有在启用 TTS 时，才额外需要：

- 宿主机可以访问 Doubao 上游服务
- `websockets`
- `unitree_webrtc_connect_leshy==2.0.7`（如果主播放链路走 `robot_webrtc`）

## 1. 基础配置

进入 `nuwax_robot_bridge` 目录，基础模式至少确认下面几组配置是对的。

### 1.1 Go2 与控制服务

```text
GO2_PROXY_TCP_HOST=0.0.0.0
GO2_PROXY_TCP_PORT=8765
GO2_DDS_IFACE=eth0
GO2_SDK_PATH=/home/unitree/unitree_sdk2_python
```

### 1.2 基础模式下建议关闭 TTS

```text
TTS_PLAYER_AUTO_START=false
TTS_LOG_BRIDGE_ENABLED=false
```

注意：

- 这两项关闭后，下面的 Doubao、播放链路、日志路径确认都可以先跳过。
- `GO2_SDK_PATH` 只会被 low-level 控制器读取。
- `go2_proxy_server.py` 的高层 SDK 导入仍要求 `unitree_sdk2py` 已经在当前 Python 环境可导入。

## 2. 安装 nuwax_robot_bridge 运行依赖

当前目录已经补了 `requirements.txt`，先安装 Python 依赖：

```bash
cd /home/ml/Deeprob/TTS_SKILL/nuwax_robot_bridge
# 首次部署或 requirements.txt 变更后执行
python3 -m pip install -r requirements.txt
```

基础模式和 TTS 模式都可以统一安装这份依赖。当前 `requirements.txt` 记录了这几类 Python 依赖：

- `websockets`
- `unitree_sdk2py` 及其 Python 侧依赖：`cyclonedds`、`numpy`、`opencv-python`
- `unitree_webrtc_connect_leshy==2.0.7`

如果你启用本地回退播放链路，再准备这些系统命令：

- `aplay`
- `ffplay`（可选）
- `wpctl` / `pactl` / `amixer`（音量兼容链路按可用性自动选）

在 Ubuntu / Debian 系常见来源通常是：

- `alsa-utils`
- `ffmpeg`
- `pulseaudio-utils` 或带 `pactl` / `wpctl` 的音频栈

另外要注意：

- `requirements.txt` 不会安装 CycloneDDS 的原生共享库；如果 `unitree_sdk2py` 导入时报 `libddsc.so` 相关错误，仍需补宿主机侧运行时。
- 如果你实际是通过本地 SDK 源码目录而不是 pip 包来提供 `unitree_sdk2py`，也仍然建议保留 `requirements.txt` 里的其余依赖。

## 3. 启动 nuwax_robot_bridge

普通启动：

```bash
cd /home/ml/Deeprob/TTS_SKILL/nuwax_robot_bridge
# 基础模式建议先在 .env 中确认：
# TTS_PLAYER_AUTO_START=false
# TTS_LOG_BRIDGE_ENABLED=false
# 首次部署或 requirements.txt 变更后执行
# python3 -m pip install -r requirements.txt
python3 go2_proxy_server.py
```

显式指定 DDS 网卡：

```bash
cd /home/ml/Deeprob/TTS_SKILL/nuwax_robot_bridge
# 如果启用 TTS，再在 .env 中打开对应配置
# 首次部署或 requirements.txt 变更后执行
# python3 -m pip install -r requirements.txt
python3 go2_proxy_server.py eth0
```

只有在启用日志桥接，且保持 `TTS_LOG_BRIDGE_REQUIRE_ROOT=true` 时，才需要按你的系统环境用具备权限的方式启动，例如：

```bash
cd /home/ml/Deeprob/TTS_SKILL/nuwax_robot_bridge
# 首次部署或 requirements.txt 变更后执行
# python3 -m pip install -r requirements.txt
sudo -E python3 go2_proxy_server.py eth0
```

启动成功后，服务会：

1. 初始化 `SportClient` / `VideoClient` / `VuiClient`
2. 如果 `TTS_PLAYER_AUTO_START=true`，自动启动：
   - `RealtimeTTSPlayer`
   - `DoubaoRealtimeClient`
   - 如果 `TTS_LOG_BRIDGE_ENABLED=true`，再启动 `LogTTSBridgeService`
3. 监听 `8765` TCP 端口

## 4. 基础联调验证

先看服务日志：

```bash
tail -f /home/unitree/nuwax_robot_bridge/go2_proxy.log
```

做一个基础控制请求，确认服务能正常收发 TCP：

```bash
printf '{"action":"balance_stand"}' | nc 127.0.0.1 8765
```

如果返回 `success=true` 或 `code=0`，说明基础控制链路已经通了。

## 5. 可选：启用 TTS

### 5.1 Doubao 与播放链路配置

只有在启用 TTS 时，才需要配置：

```text
DOUBAO_APP_ID=<你的App ID>
DOUBAO_ACCESS_KEY=<你的Access Key>
DOUBAO_RESOURCE_ID=seed-tts-1.0
DOUBAO_DEFAULT_SPEAKER=zh_female_sajiaonvyou_moon_bigtts
DOUBAO_DEFAULT_AUDIO_FORMAT=pcm
DOUBAO_DEFAULT_SAMPLE_RATE=24000
TTS_PLAYER_AUTO_START=true
TTS_PLAYER_BACKEND_MODE=robot_webrtc
TTS_PLAYER_FALLBACK_BACKEND_MODE=alsa_local
TTS_PLAYER_ROBOT_IP=<Go2 IP>
TTS_PLAYER_ROBOT_MODE=ai
TTS_PLAYER_USE_MEGAPHONE=true
TTS_PLAYER_AUTO_FALLBACK_TO_LOCAL=true
```

如果 `robot_webrtc` 缺依赖或网络不可达，且 `TTS_PLAYER_AUTO_FALLBACK_TO_LOCAL=true`，代码会尝试回退到本地 `alsa_local`。

### 5.2 可选：启用 Nuwax 日志播报

只有在你需要“从 Nuwax 日志自动播报”时，才需要确认宿主机日志路径并打开下面这些配置：

```text
TTS_LOG_BRIDGE_ENABLED=true
TTS_LOG_BRIDGE_LOG_PATH=/var/lib/docker/volumes/nuwax_home/_data/.nuwaxbot/logs/main.log
TTS_LOG_BRIDGE_REQUIRE_ROOT=true
TTS_LOG_BRIDGE_START_FROM_END=true
TTS_LOG_BRIDGE_POLL_INTERVAL_SEC=0.2
TTS_LOG_BRIDGE_QUIET_PERIOD_SEC=1.5
TTS_LOG_BRIDGE_FLUSH_INTERVAL_SEC=0.35
```

如果你没有使用 `nuwax_home` 这个卷名，或者宿主机上的实际挂载路径不同，再同步修改：

```text
TTS_LOG_BRIDGE_LOG_PATH=<你的实际宿主机路径>/.nuwaxbot/logs/main.log
```

补充说明：

- `TTS_LOG_BRIDGE_REQUIRE_ROOT=true` 时，`nuwax_robot_bridge` 会在启动桥接服务前直接检查当前进程是否为 root。
- `TTS_LOG_BRIDGE_SPEAKER` 留空时，会继承 `DOUBAO_DEFAULT_SPEAKER`。
- `TTS_LOG_BRIDGE_START_FROM_END=true` 表示启动时从日志尾部开始监听，避免把历史日志全部重播。

### 5.3 TTS 联调验证

查询 TTS 运行时状态：

```bash
printf '{"action":"tts_player_status"}' | nc 127.0.0.1 8765
```

如果只是启用 TTS 运行时，重点看：

- `running: true`
- `doubao_tts.running: true`

如果还启用了日志播报，再观察 `go2_proxy.log` 是否出现类似：

- 开始监听日志文件
- 已创建本地 Doubao 会话
- TTS流已创建

对应链路正常后，Go2 机身扬声器会开始播报。

## 常见问题

### Python 版本过低

如果日志里出现：

```text
module 'asyncio' has no attribute 'to_thread'
```

说明运行环境太老，至少要升级到 Python 3.10。

### `unitree_sdk2py` 能在 low-level 用，高层启动却报错

这是当前代码结构决定的：

- `control/low_level_controller.py` 会读取 `GO2_SDK_PATH` 并补充 `sys.path`
- `go2_proxy_server.py` 自己的高层控制导入不会这么做

所以高层控制也必须保证当前 Python 解释器能直接导入 `unitree_sdk2py`。

### `CycloneDDSLoaderException` / `libddsc.so` 报错

这不是 `requirements.txt` 缺包，而是 CycloneDDS 的原生动态库没准备好。当前项目的 Python 依赖里已经有 `cyclonedds`，但宿主机侧的 CycloneDDS 共享库和路径仍需单独配置。

### 日志桥接报 root 权限错误

当前代码在桥接启动前就会检查 `os.geteuid()`。如果你不想用 root，有两个办法：

1. 给宿主机上的 Nuwax 日志路径放宽读取权限，再把 `TTS_LOG_BRIDGE_REQUIRE_ROOT=false`
2. 继续保持 `true`，但用具备权限的方式启动 `nuwax_robot_bridge`

### `robot_webrtc` 播放链路报缺包

安装：

```bash
python3 -m pip install unitree_webrtc_connect_leshy==2.0.7
```

如果包装在另一个 Python 环境里，可以把 site-packages 路径写到：

```text
TTS_PLAYER_ROBOT_PYTHONPATH=/path/to/site-packages
```

### 改了 Nuwax 卷名后没有播报

优先检查：

1. `docker volume inspect <你的卷名>`
2. 宿主机上的真实 `main.log` 路径
3. `nuwax_robot_bridge/.env` 中的 `TTS_LOG_BRIDGE_LOG_PATH`

只要这三者不一致，日志桥接就不会读到正确文件。

## 低内存主机说明

如果宿主机内存只有 `1GB` 到 `2GB`，按官方 Docker 部署文档里的建议额外配置 swap，避免容器或 TTS 链路在高负载时被系统回收。
